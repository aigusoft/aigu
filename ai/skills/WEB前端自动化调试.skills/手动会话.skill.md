## 通过 bootstrap 信息手动执行 CDP
1. 启动浏览器并获取连接信息：
   ```bash
   ./WEB前端自动化调试.skills/tools/cdp_eval.sh --bootstrap-only > session.env
   source session.env  # 提供 devtools_host/devtools_port/chrome_pid
   ```
2. 创建新的 page target：
   ```bash
   curl -s -X PUT "http://${devtools_host}:${devtools_port}/json/new?about:blank" > target.json
   WS_URL=$(jq -r '.webSocketDebuggerUrl' target.json)
   ```
3. 发送 CDP 命令：
   ```bash
   cat <<'JSON' > enable.json
   {"id":1,"method":"Runtime.enable"}
JSON
   cat <<'JSON' > eval.json
   {"id":2,"method":"Runtime.evaluate","params":{"expression":"(() => { console.log('manual'); })()","awaitPromise":true}}
JSON
   (cat enable.json; sleep 0.2; cat eval.json; sleep 0.5) | websocat -B 65536 "$WS_URL"
   ```
4. 调试完成后，请使用 `./WEB前端自动化调试.skills/tools/cdp_eval.sh --new-chrome --bootstrap-only` 回收浏览器。
   该命令会终止当前调试进程并输出新的连接信息，确保跨 Linux/WSL 环境行为一致。

> 适合需要多轮交互或自定义 CDP 命令序列的场景。
